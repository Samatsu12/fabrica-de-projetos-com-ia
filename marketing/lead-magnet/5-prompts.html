A diferença entre quem "usa IA" e quem "produz com IA" não é habilidade técnica — é saber conduzir. Estes 5 prompts são o esqueleto dessa condução: ideia clara → primeira versão → revisão → segurança → publicação.
1

Abra o Claude Code no terminal ou no seu editor.

2

Copie o prompt e ajuste os campos em destaque vermelho.

3

Cole e envie. Leia a resposta antes de aprovar o próximo passo.

4

Avance em ordem — cada prompt prepara o terreno para o próximo.

01

Da ideia ao briefing

Quando usar: antes de pedir qualquer código — transforma ideia vaga em plano claro.

Tenho uma ideia de projeto, mas ela está bagunçada. Me ajuda a transformá-la em um briefing claro para criar com IA. Minha ideia é: [descreva sua ideia em uma frase] Organize a resposta nestes blocos: 1. Qual problema resolve. 2. Para quem serve. 3. Qual seria a primeira versão útil. 4. O que entra agora e o que fica para depois. 5. Um prompt inicial pronto para eu usar. Use linguagem simples, sem jargão técnico.
Dica: quanto mais específica a ideia, melhor o briefing. Troque "quero um site" por "quero uma página para minha clínica de estética com botão de WhatsApp e lista de serviços".
02

Primeira versão com controle

Quando usar: depois de ter o briefing — inicia a construção sem surpresas.

Vamos criar a primeira versão deste projeto. Quero algo simples, funcional e fácil de entender. Sem recursos avançados nesta etapa. Briefing: [cole o briefing gerado no Prompt 01] ANTES de criar qualquer arquivo: 1. Me mostre a estrutura que você vai usar. 2. Me explique o que cada parte faz. 3. Me diga o que ficará fora desta versão. Só comece a criar os arquivos depois da minha aprovação.
Dica: não pule a aprovação da estrutura. É aqui que você evita retrabalho — muito mais fácil mudar o plano do que refazer o código.
03

Revisão honesta

Quando usar: depois de ver a primeira versão — encontra o que não está funcionando.

Revise este projeto como se fosse um cliente comum usando pela primeira vez. Verifique: - A proposta fica clara em até 5 segundos? - O caminho para a ação principal é óbvio? - Tem texto de exemplo (lorem ipsum, "seu nome aqui")? - Alguma parte parece genérica, exagerada ou confusa? - O visual está profissional para o público-alvo? No final: lista de ajustes em ordem de prioridade.
Dica: mostre o resultado para alguém do público real antes de publicar. Uma pessoa de fora enxerga o que você deixou de ver.
04

Segurança antes de publicar

Quando usar: antes de colocar no ar — evita expor dados sensíveis por engano.

Revise este projeto procurando riscos de segurança em linguagem simples: - Tem senha, chave de API ou token escrito diretamente no código? - O arquivo .env está protegido no .gitignore? - Os dados visíveis publicamente são só os que eu quero? Liste os achados assim: 🔴 Crítico — resolva antes de publicar 🟡 Atenção — resolva em breve 🟢 OK — sem problema Para cada item crítico: me diga o passo exato de correção.
Dica: nunca coloque senhas ou chaves de API direto no código. Use variáveis de ambiente (.env) e peça ajuda ao Claude Code se não souber como configurar.
05

Publicar com tranquilidade

Quando usar: quando tudo está revisado — a reta final até o ar.

Quero publicar este projeto. Antes: 1. Rode o checklist: a página carrega, os botões funcionam, está ok no celular, sem placeholders, sem dados sensíveis no código. 2. Me recomende a forma mais SIMPLES de colocar no ar para o meu caso (Netlify Drop, Vercel, Railway...). 3. Me guie passo a passo na publicação, sem jargão. 4. Me lembre de tudo que preciso configurar antes de enviar o link para o cliente.
Dica: para páginas simples (HTML + CSS), o Netlify Drop é o mais fácil: arraste a pasta e já tem um link. Sem criar conta, sem configurar nada.
