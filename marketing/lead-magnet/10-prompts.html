10 Prompts Prontos
para criar qualquer
projeto com IA

Copie, ajuste os campos [entre colchetes] e cole no Claude Code. Cobrimos o ciclo completo: da ideia bagunçada à entrega para o cliente — sem saber programar.

A maioria das pessoas que tenta usar IA para criar coisas trava no mesmo ponto: não sabe como pedir. Estes 10 prompts resolvem exatamente isso. Estão divididos em três fases — planejar, criar e refinar — para que você saia com um projeto real, pronto para usar ou vender.
1

Abra o Claude Code no terminal ou no editor.

2

Copie o prompt e preencha os campos em destaque.

3

Cole e envie. Leia antes de aprovar o próximo passo.

4

Siga a ordem da fase 1 para a 3. Pule etapas, pague com retrabalho.

I Fase 1 Planejamento
antes de criar qualquer arquivo 2 prompts
01

Da ideia ao briefing

Quando usar: antes de tudo — transforma ideia vaga em plano claro e acionável.

Tenho uma ideia de projeto para criar com IA. Ela está na minha cabeça, mas preciso organizá-la. Minha ideia: [descreva em uma ou duas frases] Meu negócio: [o que você faz / para quem] Quem vai usar: [cliente final, funcionário, eu mesmo...] Crie um briefing com: 1. O problema real que isso resolve. 2. O que a primeira versão precisa ter (só o essencial). 3. O que pode ficar para depois. 4. Qual tecnologia simples usar. 5. Um nome para o projeto. Seja direto. Sem jargão técnico.
Dica: quanto mais específica a ideia, melhor o briefing. "Quero um site" é vago. "Quero uma página para minha clínica de estética com botão de WhatsApp e lista de serviços" é excelente.
02

Briefing completo para o Claude Code

Quando usar: depois do briefing — transforma o plano em especificação técnica sem você precisar saber programar.

Leia este briefing e me faça até 3 perguntas para entender melhor o projeto antes de começarmos. Briefing: [cole o resultado do Prompt 01] Depois que eu responder, monte o plano de execução: - Lista de telas ou arquivos que vamos criar. - O que cada um faz. - Uma coisa depende da outra? Em que ordem. - Tecnologias e por quê essa escolha. - O que NÃO vamos fazer nesta versão. Só comece quando eu confirmar o plano.
Dica: não pule a aprovação do plano. É aqui que você evita retrabalho — muito mais fácil mudar o plano do que refazer o código depois.
II Fase 2 Projetos prontos para usar
5 tipos de projeto copie o prompt do seu caso sem servidor
03

Landing page para prestador de serviço

Quando usar: para criar uma página profissional de apresentação do negócio com CTA de contato.

Crie uma landing page profissional para meu negócio. Negócio: [o que você faz] Público-alvo: [descreva em 1 frase] Ação principal: [ex: WhatsApp, formulário, agendamento] Serviços: [liste os 3–5 principais] Diferenciais: [o que te separa da concorrência] Estilo visual: [ex: clean e moderno / sóbrio e profissional] Crie: index.html + styles.css - Seções: hero, serviços, diferenciais, depoimentos (placeholder), CTA. - Mobile-first. Botão de WhatsApp fixo. Sem frameworks externos.
Dica: para publicar sem custo, use o Netlify Drop: netlify.com/drop — arraste a pasta com os arquivos e você já tem um link funcional em segundos.
04

Catálogo digital de produtos

Para negócios com produtos físicos ou serviços tabelados.

Crie um catálogo digital de produtos. Negócio: [o que você vende] Produtos: [lista com nome, preço e descrição curta] Ação: [botão WhatsApp ou formulário de pedido] Filtro/categoria: [sim ou não] Crie: index.html + styles.css + app.js - Grid de cards responsivo. - Foto placeholder em cada card. - Nome, preço, descrição, botão. - Sem banco de dados — produtos no próprio JavaScript.
05

Gerador de orçamento online

Para prestadores que vendem serviços com preço variável.

Crie uma calculadora de orçamento. Serviços: [liste com preço base] Variáveis: [o que muda o preço: área, prazo, complexidade...] Destino: [WhatsApp ou e-mail] Visual: [simples / mais elaborado] Crie: index.html + styles.css + app.js - Formulário com seleção de serviços. - Cálculo automático em tempo real. - Botão "Solicitar orçamento" que envia para WhatsApp. - Sem servidor — tudo no front.
06

CRM simples de contatos

Para quem precisa de controle de clientes sem pagar software.

Crie um CRM simples de clientes. O que registrar: nome, telefone, serviço, status, próximo contato. Status: [Interessado / Negociando / Cliente / Inativo] Usuários: [só eu / mais pessoas] Login: [sim ou não] Crie: index.html + styles.css + app.js - Tabela com busca e filtro. - Formulário para adicionar contato. - Dados no localStorage. - Exportar para CSV.
07

Dashboard com planilha CSV

Para transformar qualquer planilha em painel visual de análise.

Crie um dashboard que lê CSV. Colunas do meu CSV: [liste as colunas exatas, ex: data, vendedor, produto, valor, status] Quero ver: [total por mês, ranking, comparativo...] Quantos gráficos: [2 a 4] Crie: index.html + styles.css + app.js - Upload de CSV com drag & drop. - Cards de resumo no topo. - Gráficos (Chart.js via CDN). - Tabela de dados com filtro. - Sem servidor.
III Fase 3 Refinar e entregar
depois de criar antes de mostrar ao cliente 3 prompts
08

Revisar e melhorar qualquer projeto

Quando usar: depois de criar a primeira versão — encontra o que ainda não está pronto.

Revise e melhore este projeto que já criamos. O que o projeto faz: [explique em 2 frases] O que não está funcionando: [seja específico] O que o cliente pediu ou reclamou: [se tiver] Faça uma revisão completa: 1. Funciona corretamente em todos os dispositivos? 2. Tem texto de exemplo ou placeholder esquecido? 3. Está profissional para mostrar ao cliente? 4. O código está organizado e legível? 5. Tem algo que pode quebrar no uso real? Me dê a lista de problemas por prioridade e corrija os mais críticos.
Dica: mostre o resultado para alguém do público real antes de entregar. Uma pessoa de fora enxerga o que você, de tanto olhar, deixou de ver.
09

Simplificar quando o Claude complicou

Quando usar: sempre que a IA entregar algo maior do que o pedido — traga de volta ao essencial.

Você criou algo mais complexo do que eu precisava. Vamos simplificar. O que ficou complicado demais: [descreva o que não faz sentido] O que eu realmente preciso: [liste só o essencial] Restrições: [ex: precisa rodar sem internet / só HTML] Refaça de forma mais simples: - Remova tudo que não foi pedido explicitamente. - Use a solução mais direta possível. - Se houver escolha entre simples e sofisticado, escolha simples. - Explique o que removeu e por quê. Menos é mais.
Dica: a IA tende a adicionar recursos "por padrão". Repita este prompt quantas vezes precisar — cada rodada corta o excesso e deixa o projeto mais fácil de manter.
10

Transformar o projeto em proposta de serviço

Quando usar: depois de terminar — transforma o que você construiu em um documento profissional para vender.

Quero transformar este projeto em uma proposta comercial para o cliente. Projeto criado: [descreva o que você construiu] Cliente ou segmento: [ex: academia de bairro, salão de beleza] Valor que vou cobrar: [ou: me sugira um valor justo] Prazo de entrega: [sua estimativa] Forma de pagamento: [ex: 50% antes, 50% na entrega] Crie uma proposta profissional em HTML pronto para imprimir (A4): - Identificação do cliente e do prestador. - O que será entregue (escopo claro). - O que NÃO está incluído. - Prazo, valor e forma de pagamento. - Próximos passos. - Campo de assinatura. Tom: profissional, mas direto e sem juridiquês.
Dica: definir o que não está incluído é tão importante quanto o escopo. Evita retrabalho e mal-entendido — o cliente sabe exatamente o que comprou.
